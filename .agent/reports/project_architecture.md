# NexusHub v2: Project Architecture

## Core Tech Stack

- **Framework**: Next.js 16 (React 19, Turbopack)
- **Styling**: Tailwind CSS 4 + Framer Motion
- **Backend**: Supabase (PostgreSQL, Auth, Storage)
- **State Management**:
  - **Zustand**: Client-side UI and Auth state.
  - **React Query**: Server-side data caching and synchronization.

## Directory Structure

### `/app` (App Router)

- **`layout.tsx`**: Shell containing providers (Auth, Theme, Query, Toaster).
- **`components/screens/`**: Feature-specific views (FilesManager, Dashboard).
- **`components/effects/`**: Visual aesthetics (CarbonFiber, ParticleField).

### `/lib` (Internal Utilities)

- **`/api/`**: Supabase wrappers.
- **`/store/`**: Zustand stores.
- **`/providers/`**: Context providers (one-time initialization of clients).
- **`/types/`**: Project-wide TypeScript interfaces (`database.ts`).

### `/supabase` (Backend configuration)

- **`/migrations/`**: SQL scripts for schema building, RLS policies, and triggers.

## Security Model

- **Row Level Security (RLS)**: Enforced at the database level to ensure users can only access their own files, projects, and notes.
- **Signed URLs**: Storage objects are private by default; access is granted via temporary tokens generated by the server wrapper.
